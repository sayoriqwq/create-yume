如果你读完后只需要完成一件事，那就是：**找到与你当前目标匹配的用户文档入口。**

### 先了解项目是否适合你

读这里：

- [系统总架构](./system-architecture.md)

- 当前支持什么

## 这组文档的边界

用户文档优先解决下面这些问题：

- 这是什么
- 我现在该怎么开始
- 这个项目当前能做什么、不能做什么
- 我如何按约定协作

`docs/` 只保留稳定、当前、描述性的文档。

如果你要查看项目当前状态、持续性决策或能力契约，应分别进入 `.gsd/PROJECT.md`、`.gsd/DECISIONS.md` 与 `.gsd/REQUIREMENTS.md`，而不是把这些内容混入用户导览文档。

如果你要修改脚手架实现、模板能力或验证策略，请不要继续停留在用户文档，应改为进入执行文档：

- [执行文档路线图](../agent/roadmap.md)
