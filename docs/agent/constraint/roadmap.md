# 执行约束路线图

这份路线图写给会在仓库内执行修改、维护文档、提交代码和做验证的执行者。

如果你读完后只需要完成一件事，那就是：**根据当前任务进入正确的约束文档，而不是把所有规则都堆在主约束文件里。**

## 进入顺序

### 文档相关约束

- [文档约束](./docs.md)

适用场景：

- 新增或重写文档
- 调整文档入口
- 修改代码后判断是否需要同步文档
- 处理用户文档与执行文档分流

### 提交与 Git 相关约束

- [提交与 Git 约束](./git.md)

适用场景：

- 准备提交改动
- 判断提交信息写法
- 确认提交前最低验证动作

### 架构与实现边界约束

- [架构与实现约束](./architecture.md)

适用场景：

- 修改脚手架执行核心
- 处理 ownership-oriented architecture
- 判断某项改动是否触碰稳定边界

## 与其他执行文档的关系

- 约束文档回答“必须遵守什么”。
- [验证矩阵](../verification-matrix.md) 回答“至少要验证什么”。
- [Effect 参考路线图](../effect/roadmap.md) 回答“Effect 相关实现应该参考什么”。

不要把约束、实现说明和用户导览写成同一种文档。
