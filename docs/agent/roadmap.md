# 执行文档路线图

这份路线图写给会在仓库内执行修改、验证和维护动作的执行者或维护者。

如果你读完后只需要完成一件事，那就是：**根据当前任务，进入正确的约束文档，而不是靠猜。**

## 推荐进入顺序

1. [文档编写范式](./documentation-style-guide.md)
   - 先确认这次写的是用户文档还是执行文档。
2. [验证矩阵](./verification-matrix.md)
   - 改动前先确定最低验证集。
3. [Handlebars Helper 说明](./handlebars-helpers.md)
   - 修改模板或 helper 前先核对真实能力边界。
4. [Effect 参考路线图](./effect/roadmap.md)
   - 处理 Effect 相关实现时，先看本地参考与基线。

## 这组文档解决什么问题

执行文档优先解决下面这些问题：

- 允许改什么
- 不允许改什么
- 应该按什么顺序执行
- 哪些区域风险高
- 做完后最低要验证什么

## 和用户文档的分界

如果当前目标是“使用项目、理解项目、按仓库方式协作”，应转去：

- [用户文档路线图](../user/roadmap.md)

不要把用户入口文档当成执行手册，也不要把执行手册写成用户介绍。
