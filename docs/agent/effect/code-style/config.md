# Effect Config 基线

来源基线：

- 官方 Effect 关于配置的指导

## 官方基线

- 运行时配置应经过明确的配置边界。
- 优先使用 `Config.*` combinators，而不是临时拼装的环境变量解析。
- 原始配置解析应停留在边界处，不要在下游重复解析。
- 对敏感值使用 redaction。
- 将解析后的配置值在应用内部传递，而不是后续再反复解析原始输入。

## 适用场景

当你在做下面这些判断时，使用这份基线：

- 新增一个运行时配置项
- 审查直接访问环境变量的代码
- 判断配置解析应该放在哪里
- 处理敏感配置值

## 偏离规则

如果当前代码在配置边界之外解析原始环境值，或没有对敏感配置做 redaction，应把它视为重构信号。
