# 文档编写范式

## 读者分流

- 用户文档：服务“使用、理解、协作”
- 执行文档：服务“修改、约束执行、验证、避免偏移”

同一主题如果同时服务两类读者，应拆成两份文档，而不是混写。

## 用户文档写法

- 使用简体中文
- 描述性表达优先
- 语气友好、清楚、连续可读
- 先说目标，再说步骤，再说边界
- 优先解释“这是什么”和“我该怎么做”

## 执行文档写法

- 命令式表达优先
- 先给边界，再给流程，再给验证
- 明确约束、检查点和禁止项
- 不要写成项目介绍或背景故事

## 长期文档规则

- 标题应反映用途，而不是作者视角
- 长期文档优先描述职责、行为和边界
- 避免依赖易过期的源码路径
- 新文档必须被入口页链接，不能成为孤岛

## 自检清单

新增或重写文档时，至少确认：

- 写给谁
- 读完后对方能做什么
- 当前语气是否匹配读者
- 是否混入了另一类读者才关心的内容
- 是否已经能从正确入口被发现
