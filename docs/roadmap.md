## 当前文档分层原则

### `docs/user/`

面向仓库使用者、贡献者和第一次接触项目的人。

要求：

- 语言清晰友好
- 先解释目标，再解释操作
- 优先回答“我现在该做什么”
- 使用简体中文
- 尽量避免实现细节和内部约束语言

### `docs/agent/`

面向在仓库内执行任务的执行者或维护者。

要求：

- 强调边界、约束、前提和风险
- 优先表达流程、检查点和禁止项
- 允许使用命令式语言
- 聚焦“如何稳定地做对这件事”

### 根 `README.md`

根 README 是用户入口，不承担执行操作手册的职责。

它应该完成三件事：

1. 说明 create-yume 是什么
2. 说明用户如何开始使用
3. 把不同读者引导到正确的文档入口

## 维护规则

1. 新增文档前，先判断目标读者属于用户还是执行者。
2. 同一主题如果同时服务两类读者，应拆成两份文档，而不是混写在一页里。
3. 执行文档可以引用更严格的规则；用户文档应避免把内部实现噪音暴露给读者。
4. 长期文档优先描述职责、行为和边界，不依赖易过期的源码路径。
5. 任何新的文档入口，都应在本路线图或对应 README 中可被发现。
