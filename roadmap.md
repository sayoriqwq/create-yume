# 文档路线图

这份路线图说明 create-yume 当前文档体系的引导入口，以及不同读者应该从哪里开始。

## 根级引导文档

仓库中所有用于指示其他文档的引导文档，统一命名为 `roadmap.md`。

这样做的目的是：

- 降低入口寻找成本
- 让执行者更容易建立稳定的查找习惯
- 让不同层级的文档体系保持统一命名

## 文档分层原则

### 根 `README.md`

根 README 是用户入口。

它负责：

1. 说明 create-yume 是什么
2. 说明如何快速开始
3. 把不同读者引导到正确的 `roadmap.md`

### `docs/user/`

面向使用者、贡献者和第一次接触项目的人。

要求：

- 简体中文
- 描述性表达优先
- 可读性优先
- 先解释目标，再解释操作，再说明边界

### `docs/agent/`

面向在仓库内执行任务的执行者或维护者。

要求：

- 命令式表达优先
- 强调约束、控制、流程、检查点和禁止项
- 聚焦如何稳定完成执行

## 维护规则

1. 新增引导文档时，统一命名为 `roadmap.md`。
2. 同一主题如果同时服务用户与执行者，应拆成两份文档，而不是混写。
3. 长期文档优先描述职责、行为和边界，不依赖易过期的源码路径。
4. 新文档必须从至少一个 `roadmap.md` 或 `README.md` 可发现。
